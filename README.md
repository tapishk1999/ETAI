# 📡 SwiftRoute Revenue Intelligence Pipeline

> ML/NLP-native B2B CRM Revenue Optimisation — no external AI APIs.
> Runs fully in-browser (GitHub Pages) or as a Python CLI pipeline.

## 🤖 ML Models

| Module | Model Type | Input | Output |
|---|---|---|---|
| `sentiment_analyzer.py` | VADER (lexicon + rules) | Call transcript text | Compound score [-1,+1], pos/neg/neu, churn/anger signal counts |
| `tfidf_extractor.py` | TF-IDF (domain-biased IDF) | Transcript text | Top 12 keywords, 10 domain topic categories |
| `churn_model.py` | Logistic Regression (weighted) | 10 CRM + NLP features | Churn score 0-100, label, probability, feature importances |
| `mood_classifier.py` | Rule-based (priority cascade) | Sentiment + churn features | 6-class mood label + confidence |
| `script_generator.py` | Template engine (rule-driven) | All model outputs | 2-4 personalised conversation scripts with objection handlers |

## 🚀 Quick Start (Python)

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

> Note: Core pipeline uses only Python stdlib (re, math, csv, json).
> scikit-learn/pandas are only needed for `ChurnModel.train()`.

### 2. Run the full pipeline

```bash
# Analyse all customers in customers.csv using call_logs.csv
python src/pipeline.py

# Analyse a single customer
python src/pipeline.py --customer C005

# Analyse with a custom transcript file
python src/pipeline.py --customer C005 --log my_call.txt

# Write output to a custom path
python src/pipeline.py --output my_results.json
```

### 3. Test individual modules

```bash
python src/sentiment_analyzer.py
python src/tfidf_extractor.py
python src/churn_model.py
python src/mood_classifier.py
python src/script_generator.py
```

## 🌐 GitHub Pages Deployment

The dashboard (`docs/index.html`) is fully self-contained — all ML models
are reimplemented in vanilla JavaScript. No backend, no API keys required.

### Step 1: Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit — SwiftRoute Revenue Intelligence"
git remote add origin https://github.com/YOUR_USERNAME/swiftroute-crm.git
git push -u origin main
```

### Step 2: Enable GitHub Pages

1. Go to your repository → **Settings** → **Pages**
2. Under **Source**, select **Deploy from a branch**
3. Choose branch: `main`, folder: `/docs`
4. Click **Save**

Your dashboard will be live at:
```
https://YOUR_USERNAME.github.io/swiftroute-crm/
```

## 🔬 Model Details

### VADER Sentiment (sentiment_analyzer.py)

```
Input:  raw transcript text
Step 1: tokenise → lowercase, strip punctuation
Step 2: per-token: lookup lexicon score (-3.5 → +3.5)
Step 3: apply booster word multiplier (look-back 2 tokens)
Step 4: apply negation flip (look-back 3 tokens, × -0.74)
Step 5: churn phrase detection → penalty -2.5 per hit
Step 6: anger phrase detection → penalty -1.2 per hit (cap 5)
Step 7: normalise: compound = raw / √(raw² + 15) → [-1, +1]
```

### Logistic Regression Churn (churn_model.py)

```
linear = -2.1
       + missed_deliveries × 0.65
       + late_deliveries   × 0.30
       + open_tickets      × 0.55
       + payment_delays    × 0.45
       + (10 - nps_score)  × 0.40
       + max(0, 60-orders) × 0.04
       + call_sentiment    × -2.20
       + churn_signals     × 1.10
       + anger_signals     × 0.40
       + tier_bias         × 0.10

P(churn) = sigmoid(linear)
score    = clamp(P × 100 × 1.3, 0, 100)
```

To train with your own data:
```python
from src.churn_model import ChurnModel
import pandas as pd

model = ChurnModel()
X = pd.read_csv("your_training_data.csv")
y = X.pop("churned")  # 1 = churned, 0 = retained
model.train(X, y)     # updates weights in-place
```

## 📝 Data Formats

### customers.csv
```
customer_id, name, tier, city, orders, avg_order_value, contract_value,
join_date, last_order, open_tickets, missed_deliveries, late_deliveries,
payment_delays, nps_score, email, phone
```

### call_logs.csv
```
log_id, customer_id, date, time, speaker, line
```
Speaker must be either `AGENT` or `CUSTOMER`.

### Custom transcript format (for --log flag)
```
[2025-03-25 09:15] CUSTOMER: Your text here...
[2025-03-25 09:16] AGENT: Response here...
```
Plain text also accepted — all lines treated as customer speech.
